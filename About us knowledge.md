## **THE COMPLETE WORKFLOW**

### **PHASE 1: Gather Business Intelligence (NotebookLM)**

**Time Required:** 10-15 minutes

1. Open NotebookLM ([https://notebooklm.google.com/](https://notebooklm.google.com/))

2. Create new notebook

3. Upload target website content:

   - **Option A:** Upload all pages as PDFs/HTML  
   - **Option B:** Provide sitemap URL if NotebookLM can crawl  
   - **Option C:** Upload key pages only (homepage, service pages, about, existing blog)  
4. Open `NotebookLM_Intelligence_Questions.md`

5. Copy the entire prompt from that document

6. Paste into NotebookLM chat

7. Wait for comprehensive response (all 17 questions answered)

8. Copy NotebookLM's complete output

### **PHASE 2: Generate ALTC Recommendations (Claude AI)**

**Time Required:** 10-20 minutes

**Setup (One-Time):**

1. Create new Claude Project (or use existing ALTC project)  
2. Upload `ALTC_Fast_Track_Generator_Instructions.md` to Project Knowledge  
3. Name it something like "ALTC Fast-Track Generator"

1. Start new chat in the ALTC project  
2. Paste this prompt:

```
I have NotebookLM business intelligence for ALTC Fast-Track analysis. 
Please generate 2-3 priority Authority-Led Topic Clusters with 10-20 
specific content recommendations.

[Paste complete NotebookLM output here]
```

   - Executive Summary

## **TROUBLESHOOTING**

### **"NotebookLM can't access my website"**

- Upload pages manually as PDFs  
- Use Chrome extension to save pages as HTML  
- Provide key pages only if full site is large

### **"NotebookLM answers are too brief"**

- Ask: "Can you expand on question \[X\] with more details?"  
- Request: "Provide specific examples from the website"

### **"Information is missing from website"**

- Note which questions can't be answered  
- Provide missing info directly to Claude  
- This reveals content gaps to address\!

### **"I need this for multiple clients"**

- Keep ALTC Project open in Claude  
- Start new chat for each client  
- Copy-paste NotebookLM output per client  
- Reuse same process repeatedly

### **"Can I skip NotebookLM?"**

- If you have thorough notes about the business, yes  
- Answer the 17 questions yourself  
- Provide to Claude in same format  
- Quality of input \= quality of output

## **BEST PRACTICES**

### **For Best Results:**

1. **Be thorough with NotebookLM** \- better input \= better output  
2. **Answer Claude's clarifying questions** \- don't skip this step  
3. **Prioritize ruthlessly** \- confirm which service is highest priority  
4. **Start with Quick Wins** \- follow the implementation sequence  
5. **Track metrics** \- measure which content drives leads  
6. **Iterate based on data** \- double down on what works

### **Common Mistakes to Avoid:**

- Skipping the "which service is priority?" clarification  
- Trying to cover too many clusters at once (stick to 2-3)  
- Ignoring the Authority Anchor requirements (proof assets matter)  
- Writing content without commercial purpose (must lead somewhere)  
- Publishing randomly instead of following sequence  
- Not tracking which content drives conversions
